Starting out in R

R is both a programming language and an interactive environment for data exploration and statistics and also for computing. The easiest way to get started with R is by using the graphic interface RStudio. The main way of working with R is the console, where you enter commands and view results. RStudio surrounds this with various conveniences. In addition to the console panel, RStudio provides panels containing:

  1. Console panel for sending R commands
  2. Text editor panel where R commands can be recorded for future reference;
  3. Environment panel with a list of variables, which contain values that R has been told to save from previous commands;
  4. File manager panel;
  5. Help panel on the functions available in R;
  6. Plots panel.

Starting point: Working directory

R is always pointed at a specific directory on your computer, the working directory. The command for setting a working directory is setwd(). Check out the help panel.

# get help on get or set working directory
?getwd
Load and install packages

There are more than 10,000 of packages available to expand R. To see the list of installed packages type library(). The same command is used for loading a specific package into the workspace, making it available for the session.

# loads the package raster
library(raster)

# removes package from current workspace
detach(package:raster)

If you dont have a given package, install it with install.packages().

install.packages("raster", dependencies=T)
Variables and objects

All data structures and functions are objects in R.

# list all objects contained in the current workspace
ls()
## character(0)

We can assign data structures and functions to objects using the assignment operator (<-). RStudio helpfully shows us the variable in the Environment pane.

# assign a sequence to a object named 'mySequence'
mySequence <- seq(1,5)
mySequence
## [1] 1 2 3 4 5
ls()
## [1] "mySequence"

We can remove objects from the workspace using rm().

# removes one object from R environment
rm(mySequence)
mySequence
## Error in eval(expr, envir, enclos): object 'mySequence' not found
ls()
## character(0)
Naming objects

There is no naming convention. Even in the same package, multiple conventions might be used simultaneously. But usually, its advisable to stay consistent. Basically, there are five naming conventions to chose from:

  1. lowercase (e.g., adjustcolor);
  2. period.separated (e.g., plot.new);
  3. underscore_separated (e.g., numeric_version);
  4. lowerCamelCase (e.g., addTaskCallback)***;
  5. UpperCamelCase (e.g., SignatureMethod).
Data types and structures

R distinguishes five basic data types.

# character
speciesName <- "Laminaria ochroleuca"

# integer (subset of real numbers)
bladeSize <- 41

# numeric (allows decimals)
bladeSize <- 41.4
  
# factor
demographicTraits <- factor('juvenil',levels=c('spore','juvenil','adult'))
demographicTraits

# boolean
kelpSpecies <- TRUE
kelpSpecies

R distinguishes several objects types that differ in complexity. Vectors, matrices and arrays can only contain one data type. Data frames can contain different data types in separate columns but all columns need to have the same length. Lists can contain different object types and data types.

NA (Not Available) indicates missing values.

e.g., In a dataset with daily weather measurements over a longer period of time. On one day, the batteries of the weather station were empty and no measurements were taken. For integrity reasons, empty entries will still be stored in the data set but will be assigned the missing value NA.

mean(c(1,5,2,NA,10,4,NA,7))
## [1] NA
mean(c(1,5,2,NA,10,4,NA,7),na.rm=T)
## [1] 4.833333
Indexing

Each element of an object type is internally assigned an index (positive integer address) and you can retrieve single elements or a subset objects with these indices.

Square brackets are used for indexing. In data frames (and lists) we can access columns using the $ operator.

# produce a sequence vector
x <- seq(5,10,by=.5)

# retrieve the 3rd element of the vector x
x[3] 
## [1] 6
# use address sequences
x[3:4] 
## [1] 6.0 6.5
# use address vectors
x[c(1,4,6)] 
## [1] 5.0 6.5 7.5
# use address negative vector integers
x[-c(1,3:4)] 
## [1]  5.5  7.0  7.5  8.0  8.5  9.0  9.5 10.0
